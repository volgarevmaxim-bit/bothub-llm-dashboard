# BotHub Dashboard v2.0 — Анализ затрат LLM

## Файл
`BotHub-Dashboard-v2.html` — работает локально в браузере, без сервера.

---

## Формат входных данных (XLSX)

Ожидается выгрузка из BotHub со следующими колонками (оригинальный формат):

| Колонка | Описание | Тип |
|---|---|---|
| `Дата (UTC+0)` | Дата и время запроса | datetime |
| `CAPS` | Потрачено caps | число |
| `Бесплатные CAPS (Telegram)` | Бесплатные caps | строка (игнорируется) |
| `Ассистент` | — | — |
| `model_id` | Имя модели | строка |
| `Входных токенов` | Input токены | число |
| `Кэшированных токенов` | Cached токены | число |
| `Токенов создания кэша` | Токены создания кэша | строка (игнорируется) |
| `Выходных токенов` | Output токены | число |
| `Провайдер` | Провайдер (OpenRouter / Anthropic / …) | строка |
| `API-ключ` | Ключ API | строка (игнорируется) |
| `Пользователь` | Пользователь | строка (игнорируется) |

Загрузка — drag-and-drop или через кнопку «Выбрать файл».

## Структура дашборда

### 1. Шапка
- Название, бейдж **v2.0**
- Период: «08.07.26 — 09.07.26»

### 2. KPI (3 карточки)
- **Всего caps потрачено** — общая сумма, процент от тарифа 8 500 000
- **Стоимость итого** — в рублях по курсу 8 500 000 caps = 1 500 руб.
- **Запросов к API** — количество транзакций

### 3. Графики (сетка 3fr + 2fr + full-width)

#### a) Затраты по модели, руб. (барчарт, горизонтальный)
- Все модели, отсортированы по убыванию стоимости
- Цвета — из `PALETTE`, консистентные со всеми другими графиками
- Тултип: рубли, процент от общего, caps

#### b) Доля по типу вызова (caps) (пончик)
- Группировка моделей по семействам:
  - **Claude** — claude-*
  - **DeepSeek** — deepseek-*
  - **GLM** — glm-*
  - **TTS (озвучка)** — tts-*
  - **Whisper (распознавание)** — whisper-*
  - **Qwen** — qwen-*
  - **OpenAI** — gpt-*, o3-*, o4-*
  - **Прочее** — всё остальное
- Палитра `PALETTE_TYPES` (тёмные приглушённые тона)

#### c) Активность моделей по времени (linechart)
- Все модели (не только топ-5)
- Часовые бакеты (caps/час)
- Цвета линий — из `colorMap`, консистентны с барчартом и таблицей

### 4. Таблица «Детализация по моделям»

| Колонка | Описание |
|---|---|
| Модель | Название + цветной маркер слева |
| Провайдер | Бейдж |
| Транзакций | Количество |
| Caps (всего) | Число + визуальная шкала |
| Стоимость, руб. | 2 знака после запятой |
| Доля, % | Процент от общих caps |
| Input токены | Число или `—` |
| Output токены | Число или `—` |
| Кэш токены | Число или `—` |
| **Кэш хит, %** | **С цветовой индикацией:** |

#### Цвета «Кэш хит, %»
- 🟢 **>25%** — зелёный жирный (`#1B7A3D`) — отлично
- 🟡 **10%–25%** — жёлтый жирный (`#B8860B`) — повод для оптимизации
- 🔴 **<10% или нет кэша** — красный жирный (`#B22222`) — плохо

## Технические детали

### Парсинг XLSX
```javascript
const wb = XLSX.read(e.target.result, { type: 'array', cellDates: true });
const rows = XLSX.utils.sheet_to_json(ws, { raw: false, defval: null });
```
- `cellDates: true` — даты приходят как нативные `Date`
- `raw: false` — значения форматированные (числа как числа, даты как Date)

### Функции
| Функция | Назначение |
|---|---|
| `processRows(rows)` | Преобразует сырые строки XLSX в структуру `{summary, models, providers, timeseries}` |
| `renderAll(data)` | Точка входа — строит `colorMap`, рендерит всё |
| `renderKPIs(s)` | KPI-карточки |
| `renderBarChart(models)` | Горизонтальный барчарт |
| `renderCallTypeDonut(models)` | Пончик по типам вызовов |
| `renderTimeChart(ts)` | Линейный график по времени |
| `renderTable(models)` | Таблица детализации |
| `buildColorMap(models)` | Строит `{modelName → color}` для консистентности |
| `classifyModel(name)` | Определяет тип вызова по имени модели |
| `badgeClass(p)` | CSS-класс для бейджа провайдера |
| `fmtNum(n)` | Форматирование чисел (1.5K, 2.3M) |
| `parseSumma(val)` | Парсинг суммы (не используется в v2, но оставлен для обратной совместимости) |
